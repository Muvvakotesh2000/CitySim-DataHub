CitySim Workflow Diagram

Step 1: Data Generation

Simulated data is generated using Python scripts. Data types include Environment, IoT Sensor, Public Services, Traffic, and Utilities data.

Step 2: AWS Infrastructure Setup (Terraform)

Terraform scripts create and configure AWS resources including S3 buckets, CloudWatch log groups, security groups, subnets, and a VPC.

Step 3: Apache Kafka Data Streaming

Data is streamed via Kafka using Python producers. Kafka topics are created for traffic, utilities, public services, environment, and IoT sensors.

Step 4: Kafka & Apache Spark Processing

Kafka data is consumed by Apache Spark for schema validation, time-based aggregations, data cleaning, and transformation. Processed data is stored in S3.

Step 5: AWS S3 Storage

Raw and processed data are stored in separate S3 buckets. The raw bucket holds unprocessed data from Kafka while the processed bucket holds Spark outputs.

Step 6: Flattening the Processed Data

Every five minutes, the processed data is flattened to simplify its structure for downstream analytics and visualization.

Step 7: Pushing Data to OpenSearch

An AWS Glue job pushes the flattened data from the processed S3 bucket to OpenSearch. Previous indexes are deleted and new ones created.

Step 8: Real-Time Data Dashboards

A Python script converts OpenSearch JSON objects to CSV format, which is then connected live to Tableau for dashboard visualization.

Step 9: Merging Data (AWS Glue Job)

Data generated over the last hour is merged via an AWS Glue job. Nightly merging at 11:30 PM creates comprehensive daily files in both raw and processed buckets.

Step 10: Flattening Raw Data (AWS Glue Job)

At 11:40 PM, a Glue job flattens the large raw data files using batch processing, outputting a flattened CSV file stored in an output bucket.

Step 11: Pushing Flattened Raw Data to RedShift

A Lambda function, triggered by an EventBridge rule at 11:45 PM, loads the flattened CSV into RedShift using the COPY command based on predefined schemas.

Step 12: Deleting All Data

At 12 PM, a Lambda function (via EventBridge) deletes all data from the S3 buckets to clear the slate for the next dayâ€™s data ingestion.
